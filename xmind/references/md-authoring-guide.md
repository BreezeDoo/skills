# Markdown Authoring Guide — 生成可高质量转换为 .xmind 的 md

> 用途：当你（LLM）需要**从零生成**一份思维导图，而不是转换用户已有的 md 时，先读本指南。目标是产出一份结构清晰、能被 `xmind-tool convert foo.md -o foo.xmind` 无歧义解析的 md。

## 核心原则

**写标题树，少写列表。** 标题（`#`/`##`/`###`）能表达 sheet 边界和多层嵌套；纯列表只能挂在一个父节点下、且缩进容错差。混用可以，但骨架用标题。

## 结构骨架（最重要）

```
# 仿真资源管理          ← H1 = 一个 sheet 的标题（必须有，每张图至少一个 H1）

## 仿真资源管理          ← H2 = 这张图的中心主题（root topic）

### 文档：杜宇航          ← H3 = root 的子节点
#### 《脚本开发规范》      ← H4 = 上一行的子节点
##### 现有脚本规范        ← H5 = 再深一层
```

规则：
- **每个 `#` H1 开一张新 sheet**。多张图就写多个 H1。
- **每个 H1 后紧跟一个 `##` H2** 作为这张图的中心主题。H1 只是 sheet 名，不进树。
- **H3+ 是子节点**，深度 = 标题级别 − 2（H3→root 的子节点，H4→H3 的子节点…）。最深支持 H6（4 层子节点）。再深就用列表。
- **第二个 `##` H2 在同一 H1 下** = 开新 sheet（H2 同时作 sheet 名和 root）。所以一张图内别误写第二个 H2，除非你真想开新图。

## 何时用列表 `-`/`*`

列表项是「当前节点的子节点」，用 4 空格缩进表示层级：

```
#### 《C2及行为规范》
- Capability           ← 《C2及行为规范》的子节点
    - 战术层           ← Capability 的子节点（4 空格缩进）
    - 战役层
- C2层级结构           ← 回到《C2及行为规范》的子节点
```

适合：叶子节点多、层级浅、不想数 `#` 个数时。**不适合**做整图骨架——列表无法表达「换 sheet」。

## 节点附加信息（写在节点标题之后）

| 想表达 | md 写法 | 说明 |
|---|---|---|
| 备注 | 标题下一行起 `> 备注`，多行连续 `>` | blockquote，多行自动合并为一条 note |
| 标签 | 标题下一行 `**Labels:** 标签1, 标签2` | 逗号分隔 |
| 标记 | 标题下一行 `**Makers:** flag, task` | XMind marker 名 |
| 链接 | 标题下一行 `[描述](https://url)` | 方括号文字会被丢弃，只取 url |
| 标注框 | 标题下一行 `**Callout:** 提示文字` | 气泡标注 |
| 概要框 | 与被概括的兄弟**同级**、标题写 `#### [概要] 概要标题`（`#` 数与兄弟一致） | 自动括住父节点下它之前的全部兄弟 |
| 游离主题 | 与某节点**同级**、标题写 `#### [游离] 浮动主题`（`#` 数与同级一致） | 浮动节点，不挂在父树上 |

**示例：带备注+标签的节点**
```
##### 现有脚本规范
> 必须遵守变量作用域规范
**Labels:** 规范, 必读
```

## 会被丢弃的东西（不要在 md 里指望它们）

| 特性 | 原因 | 替代方案 |
|---|---|---|
| 关联线（relationships） | md 是线性文本，无法表达两点连线 | 导出后用 JSON 通道补 `relationships`；或接受 md 图无线 |
| 视觉样式（加粗/颜色/字号） | XMind 加粗走 `style.properties`，非文本符号 | 标题里的 `**bold**`/`` `code` `` 会被**剥离成纯文本** |
| 图片 | 图片是二进制，md 不携带 | 导出后在 XMind 里手动插图，或用 JSON 通道 |
| 概要框的范围括住哪几个 | md 只能「括住它前面所有兄弟」，不能指定任意区间 | 复杂范围用 JSON 通道的 `range` |

## `## 关联` 段是什么？我能用它建关联线吗？

**不能。** xmind→md 会把已有关联线渲染成 sheet 末尾的只读注脚段（`## 关联` + `- A ↔ B`），让你看 md 时知道有哪些关联。但 md→xmind **整段丢弃**它——关联线不在 md 通道创建。要在 md 图里建关联线，转成 .xmind 后用 JSON 通道补。

## 完整范例

```
# 项目规划

## 2025 Q4 产品路线图

### 后端
#### API 重构
> 兼容旧接口到年底
**Labels:** 技术债, P1
- 用户服务拆分
    - 鉴权独立
    - 资料服务化
- 订单服务
#### 数据库迁移
[迁移文档](https://wiki.example.com/db-migration)

### 前端
#### 组件库升级
**Makers:** flag
#### 性能优化
#### [概要] 首屏 < 1s
```

转成 xmind 后：1 张图「项目规划」，中心主题「2025 Q4 产品路线图」，后端/前端两个分支，API 重构带备注和标签，数据库迁移带链接，前端下挂概要框「首屏 < 1s」（括住「组件库升级」+「性能优化」两个兄弟）。

## 生成流程建议

1. 先确定**几张图**（几个 H1）和每张图的**中心主题**（H2）。
2. 用 H3/H4/H5 搭 2-4 层骨架，叶子层可用列表。
3. 给关键节点补备注（`>`）和标签（`**Labels:**`）。
4. 转换：`xmind-tool convert plan.md -o plan.xmind`（默认带主题，视觉即开即用）。
5. 如需关联线/图片/复杂样式，转完后用 JSON 通道补。

## 反模式（避免）

- ❌ 整图只用 `-` 列表，没有 `#` 标题 → 无法表达多 sheet，缩进容错差
- ❌ 在标题里写 `**加粗**` 想让节点加粗 → 被剥离成纯文本，XMind 里不显示加粗
- ❌ 用 `## 关联` 段想建关联线 → 整段被丢弃，建不出来
- ❌ 一个 H1 下写多个 `##` H2 当子节点 → 每个都会开新 sheet
- ❌ 缺 H1 直接 `##` 开头 → 被塞进名为 "Imported" 的默认 sheet
